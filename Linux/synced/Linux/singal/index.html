Linux Signal

SIGINT (Signal Interrupt)

  • 信号编号: 2
  • 默认行为: 终止进程
  • 触发方式:
    • 用户按下 Ctrl+C（终端中断）
    • 使用 kill -INT <pid> 或 kill -2 <pid>
  • 典型用途:
    • 用户主动中断程序
    • 允许程序进行清理工作（可被捕获处理）
    • 通常用于交互式程序的优雅退出

SIGTERM (Signal Terminate)

  • 信号编号: 15
  • 默认行为: 终止进程
  • 触发方式:
    • 使用 kill <pid>（默认发送 SIGTERM）
    • 使用 kill -TERM <pid> 或 kill -15 <pid>
  • 典型用途:
    • 系统正常关闭时终止进程
    • 管理员手动终止进程
    • 进程管理工具（如 systemd, supervisor）停止服务
    • 允许程序进行清理（可被捕获处理）

others

  • SIGKILL (9): 强制立即终止，不可被捕获或忽略
  • SIGSTOP (19): 暂停进程（不可捕获）
  • SIGHUP (1): 终端断开或控制进程结束

信号捕获原则

  1. 程序应该捕获 SIGTERM 进行清理，因为这是系统正常关闭的方式
  2. SIGINT 通常也应该被捕获，提供友好的用户交互
  3. 不要忽略 SIGTERM，除非有特殊原因
  4. 无法捕获 SIGKILL，因此不能依赖它进行清理
